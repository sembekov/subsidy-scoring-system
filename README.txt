Методология Merit-Based Scoring для распределения субсидий в АПК
1. Назначение и область применения
Настоящая система предназначена для перехода от формального принципа распределения государственных субсидий («первый подал — первый получил») к объективному, метрико-ориентированному механизму принятия решений.

Система анализирует исторические данные о заявках и формирует рейтинг получателей (акиматов, районов, сельхозпроизводителей) на основе их реальной эффективности, стабильности и результативности использования бюджетных средств.

3. Характеристика исходных данных
Исходный датасет представляет собой выгрузку из информационной системы субсидирования (subsidy.plem.kz) и содержит следующие ключевые поля:

Поле	Описание
Дата поступления	Момент подачи заявки (временная метка)
Область / Регион	Административно-территориальная единица
Акимат	Орган, уполномоченный на распределение субсидий
Направление деятельности	Отрасль животноводства (скотоводство, птицеводство и др.)
Тип субсидии	Конкретная программа субсидирования
Статус заявки	Исполнена, одобрена, отклонена, отозвана
Норматив	Базовая ставка субсидирования (тенге)
Причитающаяся сумма	Итоговый объём выплаченных средств
Район хозяйства	Муниципальное образование заявителя
Особенности данных:

Заголовок не фиксирован → автоматическое определение позиции заголовка

Наличие пропусков и «шумовых» значений (текст в числовых полях)

Неоднородные форматы чисел (разделители: пробелы, запятые, точки)

4.3. Инжиниринг признаков
Система генерирует следующие производные признаки:

Признак	Формула / Смысл
Efficiency	Сумма / Норматив — эффективность использования бюджетного рубля
Success Rate	Доля исполненных заявок от всех поданных
Stability	Обратная величина стандартного отклонения эффективности
Activity	Количество поданных заявок
Scale	Общий объём полученных субсидий
Региональное позиционирование	Отклонение эффективности заявителя от среднерегиональной
5. Модельный подход
Используется гибридная архитектура, сочетающая машинное обучение и экспертные правила.

5.1. Предиктивная модель
Параметр	Значение
Алгоритм	Random Forest
Целевая переменная	Будущая эффективность (Efficiency)
Метрика оценки	R² (коэффициент детерминации)
Валидация	5-кратная кросс-валидация
Результаты текущей модели:

R² ≈ -0.21 (отрицательное значение)

Интерпретация: объём и качество данных недостаточны для стабильного прогнозирования; наблюдается высокая вариативность и слабая предсказуемость.

5.2. Rule-based scoring (Fallback-механизм)
При низком качестве предсказаний система переключается на правило-ориентированный скоринг:

text
Score = 0.30 × Efficiency
      + 0.20 × Amount
      + 0.20 × Success_Rate
      + 0.15 × Stability
      + 0.15 × Regional_Position
Данный подход гарантирует:

Устойчивость — система работает даже при плохих данных

Интерпретируемость — каждый компонент балла объясним

Практическую применимость — может быть использован уже сейчас

6.1. Рейтинг заявителей
Каждому акимату / району присваивается интегральный балл (0–100), отражающий его приоритетность для получения субсидий.

6.2. Топ-кандидаты
Формируется шорт-лист приоритетных получателей с детальной аналитикой:

Акимат	Балл	Эффективность	Success rate	Стабильность
г. Шымкент	92	Высокая	Высокая	Стабильный
Туркестанская обл.	67	Средняя	Низкая	Вариативный
Акмолинская обл.	85	Высокая	Низкая	Стабильный
6.3. Интерпретация и рекомендации
Для каждого кандидата система формирует:

Сильные стороны (драйверы высокого балла)

Слабые стороны (факторы, снижающие балл)

Рекомендации по улучшению

Пример:

Акмолинская область
✅ Сильные стороны: высокая эффективность использования средств, стабильность
⚠️ Слабые стороны: низкий success rate (доля исполненных заявок)
💡 Рекомендация: усилить контроль качества подаваемых заявок, повысить документарную дисциплину.

8. Ограничения и пути улучшения
8.1. Текущие ограничения
Ограничение	Описание
Качество данных	Шум, пропуски, неоднородные форматы
Слабая предсказательная сила	Отрицательный R² модели
Отсутствие внешних факторов	Климат, тип хозяйства, экономические условия не учитываются
8.2. Рекомендации по развитию
Обогащение данных — интеграция с внешними источниками (метеоданные, кадастровая информация, финансовые показатели хозяйств)

Накопление истории — увеличение временного горизонта данных для повышения качества прогнозов

Кастомизация модели — разработка отдельных моделей для разных направлений животноводства

Внедрение онлайн-режима — подключение к API информационной системы для расчёта скоринга в реальном времени

9. Практическая ценность
Разработанная система позволяет:

Задача	Решение
Приоритизация заявок	Автоматический расчёт балла для каждой заявки
Распределение бюджета	Обоснованное выделение средств наиболее эффективным получателям
Выявление неэффективных получателей	Ранжирование с выделением аутсайдеров
Мониторинг регионов	Сравнительный анализ по областям и районам
Поддержка управленческих решений	Прозрачная, интерпретируемая метрика для комиссий
Ключевое преимущество:
Решения становятся объективными, воспроизводимыми и защищёнными от субъективизма, что повышает доверие к системе распределения государственных субсидий.

10. Заключение
Представленная система скоринга является работоспособным прототипом, который уже сегодня может использоваться для:

предварительной фильтрации заявок

формирования рекомендаций для распределительных комиссий

выявления проблемных зон в региональном разрезе

При накоплении большего объёма данных и их обогащении внешними источниками предсказательная способность модели может быть существенно повышена. Однако уже в текущем виде система обеспечивает переход к метрико-ориентированному распределению субсидий, что является значительным шагом в повышении эффективности государственной поддержки АПК.
